# CRM智能数据分析平台 - 启动与开发说明

## 一、目录约定
- 后端目录：`D:\pyCharmProjects\workSpace02`
- 前端目录：`D:\webStormProjects\workSpace05`
- 后端 Conda 环境：`D:\pyCharmProjects\workSpace02\.conda\crm_ai`

## 二、已实现能力（与你截图对齐的核心）
- 左侧分组导航：首页、线索管理、商机管理、履约管理、客户管理、AI问答、开发测试
- 看板页：指标卡、漏斗图、阶段进度堆叠条、BU预测条形图（接口异常时自动降级为演示数据）
- 线索管理页：全部线索、线索录入、线索验证、立项资料、立项评审（支持筛选 + 新增 + 编辑）
- 商机管理页：全部商机、提案准备、提案评审、客户定标（支持筛选 + 新增 + 编辑 + 阶段推进校验）
- 履约管理页：全部履约、赢单阶段、开发阶段、量产阶段、销售订单（支持筛选 + 新增 + 编辑）
- 客户管理页：搜索、等级筛选、新增、编辑
- 全模块支持分页、排序、CSV导出
- 审计日志页：按模块/操作人查看关键操作轨迹（经理及以上可见）
- 看板钻取：支持按负责人/阶段/客户维度联动明细
- 角色权限：销售/经理/总监/管理员，不同角色的数据范围与可执行动作不同
- AI问答页：真实调用大模型（`gpt-4o-mini`，你给的 key/base_url）
- 并发测试页：可配置线程数与任务数，演示锁和高并发

## 三、后端技术（轻量化）
- FastAPI + SQLite + SQLAlchemy
- 轻量缓存：内存 TTL 缓存（无 Redis 依赖）
- 并发：ThreadPoolExecutor + Lock
- AI：langchain-openai（真实联网调用）

## 四、已使用设计模式
- Repository Pattern：`CustomerRepository`、`OpportunityRepository`、`LeadRepository`、`FulfillmentRepository`、`DashboardRepository`
- Service Pattern：`DashboardService`、`AIService`、`ConcurrencyService`
- Facade Pattern：`AIService` 对上层隐藏大模型调用细节
- Strategy 思想：`ConcurrencyService` 可扩展不同并发策略

## 五、日志格式
- 全部按你要求：`类名.方法名 >>> 日志信息`
- 示例：`DashboardService.get_overview >>> 缓存未命中，执行数据库聚合`

## 七、手动启动步骤（你要求的文档重点）

### 1）启动后端
在 PowerShell 执行：

```powershell
cd D:\pyCharmProjects\workSpace02
chcp 65001
$env:PYTHONUTF8 = "1"
.\.conda\crm_ai\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

后端地址：`http://127.0.0.1:8000`

### 2）启动前端
新开一个 PowerShell 执行：

```powershell
cd D:\webStormProjects\workSpace05
npm run dev
```

前端地址：`http://localhost:5173`
> 说明：若 5173 端口被占用，Vite 会自动切换为 5174/5175 等可用端口。

### 3）访问页面
- 看板页：`http://localhost:5173/dashboard`
- 线索页（全部线索）：`http://localhost:5173/leads/all`
- 商机页（全部商机）：`http://localhost:5173/opps/all`
- 履约页（全部履约）：`http://localhost:5173/fulfill/all`
- 客户页：`http://localhost:5173/customers`
- AI页：`http://localhost:5173/ai-chat`
- 开发测试页：`http://localhost:5173/concurrency`

## 八、接口清单
- `GET /api/health` 健康检查
- `GET /api/dashboard/overview` 看板数据
- `GET /api/dashboard/drilldown` 看板钻取明细
- `GET /api/customers` 客户列表
- `POST /api/customers` 新增客户
- `PUT /api/customers/{customer_id}` 编辑客户
- `GET /api/customers/export.csv` 导出客户
- `GET /api/leads` 线索列表（支持 status/keyword 过滤）
- `POST /api/leads` 新增线索
- `PUT /api/leads/{lead_id}` 编辑线索
- `POST /api/leads/{lead_id}/convert-to-opportunity` 线索转商机
- `POST /api/leads/import-csv` 线索CSV导入
- `GET /api/leads/export.csv` 导出线索
- `GET /api/opportunities` 商机列表
- `POST /api/opportunities` 新增商机
- `PUT /api/opportunities/{opp_id}` 编辑商机
- `POST /api/opportunities/{opp_id}/advance-stage` 商机阶段推进（带校验）
- `GET /api/opportunities/export.csv` 导出商机
- `GET /api/fulfillments` 履约列表（支持 stage/keyword 过滤）
- `POST /api/fulfillments` 新增履约
- `PUT /api/fulfillments/{row_id}` 编辑履约
- `GET /api/fulfillments/{row_id}/milestones` 查询里程碑
- `POST /api/fulfillments/{row_id}/milestones` 新增里程碑
- `PUT /api/fulfillments/milestones/{milestone_id}` 编辑里程碑
- `GET /api/fulfillments/overdue-milestones` 逾期预警列表
- `GET /api/fulfillments/export.csv` 导出履约
- `GET /api/system/audit-logs` 审计日志分页查询（经理及以上）
- `GET /api/system/audit-logs/export.csv` 审计日志导出
- `POST /api/ai/chat` AI问答
- `GET /api/system/concurrency-test?workers=20&tasks=100` 并发测试

## 十一、角色切换说明（前端）
- 页面右上角可设置用户名和角色，系统会自动把身份通过请求头传给后端。
- 角色说明：
  - `sales`：只能查看本人数据
  - `manager`：可看全量数据，可执行线索转商机、阶段推进、CSV导入、里程碑维护
  - `director/admin`：同样具备高级权限
